    JavaValidationRendererSettingsProto
    Builder
    Settings consumed by `io.spine.tools.validation.java.JavaValidationRenderer`.

The Validation Gradle plugin writes this message to the Spine Compiler
settings directory before the renderer runs, so user-configured values
from the `validation { java { ... } }` DSL block reach the renderer
running in the Spine Compiler subprocess.

    Builder
    Per-kind toggles for suppressing warnings emitted by the Java target
of the Validation Compiler.

Each flag is independent. A flag set to `true` suppresses the
corresponding warning; the proto3 default `false` keeps the warning
emitted, which matches the renderer's "warn unless told otherwise"
behavior when no settings are present at all.
    Protobuf type spine.validation.java.SuppressWarnings
